# Golden harness

The cross-language golden invariant: for a given `CompileSpec`, every binding —
Rust core, the CLI, and all ten language bindings — produces the **byte-identical
manifest**. This directory pins that manifest for a fixed set of specs so any
drift (in codegen, templates, hashing, or a binding) is caught immediately.

## Layout

- `candles/{BTCUSDT,ETHUSDT}.csv` — deterministic OHLCV candles for the
  `embed_data` specs (42 bars each).
- `specs/*.json` — five `CompileSpec` fixtures covering the target and option
  matrix (wasm / binary / no_std, with and without embedded data).
- `expected/*.json` — one blessed `Manifest` per spec: the exact
  `compile { dry_run: true }` output, committed byte-for-byte.

## What the golden is (and is not)

The golden is the **manifest**, not the built artifact bytes. The manifest pins
the generated project: every generated file's path and SHA-256, the spec hash,
the compiler version, and the aggregate `project_hash`. It is fully deterministic
and needs no toolchain.

The built artifact bytes are **not** golden — a byte-reproducible binary needs a
pinned, reproducible build toolchain, which is out of scope here. `dry_run: true`
stops before invoking `cargo`, so the manifest is all that is compared.

## Candle formula

Both CSVs are generated by a fixed integer-rounded sine path (stable across
platforms), with `time = 1700000000 + 3600 * i` for `i` in `0..42`:

```
close  = base + round(amp * sin(i * 0.25 + phase))
open   = previous close (open[0] = close[0])
high   = max(open, close) + 10
low    = min(open, close) - 10
volume = 1000 + i
```

- `BTCUSDT`: `base = 30000`, `amp = 2000`, `phase = 0`.
- `ETHUSDT`: `base = 2000`, `amp = 150`, `phase = 1`.

## Blessing (regenerating the expected manifests)

Never edit `expected/*.json` by hand. To re-bless after an intentional codegen
change, run from the repository root (so `embed_data` CSV paths resolve):

```bash
cargo build -p wickra-compile --release
for s in sma_cross ema_trend rsi_reversion no_std_blink binary_daemon; do
  target/release/wickra-compile --spec "golden/specs/$s.json" --manifest \
    > "golden/expected/$s.json"
done
```

Review the diff carefully — a change here means the manifest, and therefore every
binding's output, changed.
